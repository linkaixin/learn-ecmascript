DNS解析过程
www.jsplusplus.com.
1. 先查询DNS本地服务器，是否有这个域名的IP地址，如果缓存中存在就直接返回
2. DNS本地服务器不存在，就寻找根服务器是否存在这个域名的IP地址，根服务器会让去寻找.com域服务器
3. .com域服务器中是否有这个域名的IP地址，没有的话就会返回运营商的域服务器地址。
4. 到该运营商的域服务器中去寻找该域名的IP地址，返回到本地的DNS服务器，存入DNS服务器缓存
IP Internet Protocol Address 互联网协议地址、IP地址
分配给用户上网使用的互联网协议，用户用IP协议地址获取服务
分类：IPv4、IPv6、其他
IPv4：192.168.0.1 长度32、4个字节、十进制表示
IPv6：2001:0db8:85a3:0000:0000:8a2e:0370:7334 长度128、16个字节、16进制表示
IPv6的优势：
1. IPv6地址空间更大；
2. 路由表更小；
3. 组播支持以及对流支持增强；
4. 更高的安全性；
5. 对自动配置的支持；

范围：0 – 65535

TCP：Transmission Control Protocol 传输控制协议
特点：面向连接（收发数据前，必须建立可靠的连接，数据必须准确无误的收发）
建立连接基础：三次握手
应用场景：HTTP请求、FTP文件传输、邮件收发
优点：稳定、重传机制、拥塞控制机制、断开连接
缺点：速度慢、效率低、占用资源、容易被攻击（三次握手->DOS、DDOS攻击）
TCP/IP协议组：提供点对点的连接机制，制定了数据封装、定址、传输、路由、数据接收的标准。
UDP：User Data Protocol 用户数据报协议
特点：无连接（不可靠的协议，无状态传输机制）,无需确保通讯质量且要求速度快、无需确保信息完整
应用场景：消息收发、语音通话、直播、QQ
优点：安全、快速、漏洞少
缺点：不可靠、不稳定、容易丢包、遭受UDP flood攻击
总结：只要目的源地址、端口号、地址、端口号确定，则可以直接发送信息报文，但不能保证一定能收到或收到完整的数据。
HTTP: HyperText Transfer Protocol 超文本传输协议
定义：客户端和服务器端请求和应答的标准，用于从WEB服务器传输超文本到本地浏览器的传输协议
HTTP请求：按照协议规则先向WEB服务器发送的将超文本传输到本地浏览器的请求
HTTPS: HyperText Transfer Protocol Secure 超文本传输安全协议
定义：HTTP的安全版，安全基础是SSL/TLS。
SSL：Secure Sockets Layer  安全套接层
TLS：Transport Layer Security 传输层安全
为网络通信提供安全及数据完整性的一种安全协议，对网络连接进行加密
区别
1. HTTP是不安全的，可以通过监听或中间人等攻击手段，获取网站的账户信息和敏感信息，而且HTTPS是密文传输，可防止被攻击
2. HTTP协议的传输内容都是明文，直接在TCP连接上运行，客户端和服务器都无法验证对方身份
3. HTTPS协议的传输内容都被SSL/TLS加密，且运行在SSL/TLS上，SSL/TLS运行在TCP连接上，所以数据传输是安全的。
建立TCP连接
标志位：数据包。
SYN：Synchronize Sequence Numbers 同步序列编号
ACK：Acknowledgement 确认字符
状态：
LISTEN：服务端侦听TCP端口的连接请求
SYN-SENT：在发送连接请求后等待匹配的连接请求
SYN-RECEIVED：在收到和发送一个连接请求后等待对连接请求的确认
ESTABLISHED：代表一个打开的连接，数据可以传送给用户
建立TCP三次握手
第一次握手：客户端向服务器发送连接请求，发送标志位SYNJ和序号200，进入SYN_SENT状态等待服务器确认
第二次握手：服务器确认该数据包已收到，并且向客户端发送标志位ACK J + 1、标志位SYNK、序号500和确认序号201，进入SYN_REVD状态等待客户端确认
第三次握手：客户端建立连接进入ESTABLISHED状态，向服务器发送标志位ACK K+1和发送序列201、确认序列501，确认客户端已收到建立连接确认，服务器收到ACK K+1后，服务器建立连接，进入ESTABLISHED。
C/S架构与B/S架构
C/S Client/Server
将应用程序安装在客户端电脑中，由服务端提供客户端程序所需要的数据，例如QQ、微信等应用程序
优点：界面与操作丰富、安全性高、响应速度快。
缺点：通常用于局域网、需安装特定应用程序或使用特定硬件、维护成本高。
B/S Browser/Server
利用WEB浏览器呈现客户端程序界面，由服务端提供客户端程序所需要的数据
优点：无需安装客户端程序与特定硬件、多客户访问、交互性强、无需升级客户端
缺点：跨浏览器兼容性差、功能性相对较弱、设计成本高、安全性弱、功能性弱
